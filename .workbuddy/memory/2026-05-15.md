- 问题：AI 回复中出现 `@image#1:image.png`，但右侧游戏未更新
- 修复内容：
  1. `extractHtmlCode` 函数重写：使用更可靠的方式匹配代码块（从 `\`\`\`html` 后开始，匹配到最后一个 `\`\`\``），支持未闭合代码块兜底提取
  2. 新增 `cleanImageMarkers` 函数：过滤 AI 可能输出的 `@image#...` 图片占位符
  3. `extractTextOnly` 增加过滤 `@image#...` 标记，避免聊天界面显示垃圾内容
  4. `TEACHER_SYSTEM_PROMPT` 新增两条规则：
     - 绝对禁止输出图片引用或图片占位符
     - 代码块必须正确闭合（以 `\`\`\`html` 开头，以 `\`\`\`` 结尾）
- 2026-05-15 后续修复：
  1. 修复正则表达式语法错误：将 `/@image#\d+:[^\s\n]+/g` 改为 `new RegExp("@image#\\d+[:：][^\\s\\n]+", "g")`，避免 Next.js 编译时的"Unterminated regexp literal"错误
  2. 增强图片占位符检测：支持中英文冒号（`:` 和 `：`）
  3. 强化系统提示词：添加输出格式强制要求，确保 AI 在单独段落中输出完整 HTML 代码块
  4. 验证构建通过，无 TypeScript 错误

## 修改游戏功能3个致命缺陷修复
- 缺陷1（最致命）：doSend 发给 API 的 messages 来自前端显示状态，AI 回复中的代码已被 extractTextOnly 清除，AI 看不到之前的游戏代码
  - 修复：在 apiMessages 末尾追加一条包含完整 htmlCode 的 assistant 消息，让 AI 能看到当前代码
- 缺陷2：currentCode 注入依赖关键词正则匹配（修改|更新|改成等），小学生常用表达如"帮我把按钮变大"可能匹配不到
  - 修复：去掉正则判断，只要 currentCode 存在就以"修改模式"强提示注入系统提示词
- 缺陷3：自动重试机制只发一条无上下文的消息"请输出代码"，AI 无法正确理解
  - 修复：重试时带上完整对话上下文 + 当前代码
