# 第1节：我是游戏设计师

## 教学目标
- 学生能说出游戏由「规则」和「画面」组成
- 学生能完成基础情况调查
- 学生能用「如果___，就___」写 2 条游戏规则
- 学生能选择游戏类型并绘制设计图

## 教学脚本（20分钟）

### [0-3分钟] 导入：什么是游戏设计师

```
👨‍🏫 老师：（展示3个AI生成的游戏画面）
"同学们，你们看！这几个游戏都是和你们一样大的小学生做的。
今天开始，你们每个人都要成为一名游戏设计师。
你们觉得一个游戏最需要什么？"

👦 学生A："需要有人玩"
👧 学生B："需要过关"
👧 学生C："要有分数"

👨‍🏫 老师："都对！一个游戏最重要的就是——规则。
规则告诉我们：碰到什么会怎样，怎么算赢，怎么算输。"
```

### [3-8分钟] 示范：平台操作演示

```
👨‍🏫 老师：（投影演示，逐步操作）
"现在看我电脑屏幕。我会一步步演示怎么做。

第一步：进入「游戏构思」。
我们会先做一个小调查，回答5个问题。

第二步：选择你想做的游戏类型。
（展示游戏类型列表：接东西、躲避、跑酷、迷宫、对战）

第三步：画设计图。
在这里，你可以画出你的游戏画面。
（展示5种预设画布背景，选择"星空"背景）

第四步：写规则。
这是最重要的！用这个句式：
'如果___，就___'

例子：
✅ 如果接到星星，就得一分
✅ 如果碰到障碍物，就扣一条命

说清楚条件和结果，这就是规则。"
```

### [8-15分钟] 理解检查：说出我的规则

```
👨‍🏫 老师："现在每个人在脑子里想一想：你要做什么游戏？
给你30秒。"

（30秒思考时间）

👨‍🏫 老师："好，谁来告诉我——你的游戏有一条什么规则？"

👦 学生1："我想做接金币的游戏。如果接到金币就加分。"
👨‍🏫 老师："非常好！条件是什么？结果是什么？"
👦 学生1："条件是接到金币，结果就是加分。"

👧 学生2："我做跑酷游戏。如果撞到石头就扣一条命。"
👨‍🏫 老师："规则很清晰！撞到石头=条件，扣命=结果。"

👦 学生3："我做对战游戏。如果打中敌人就赢了。"
👨‍🏫 老师："可以，再想想能不能加一条规则让它更好玩？"

（让3-4个学生口头表述，确保全班理解）
```

### [15-17分钟] 提示：常见问题预防

```
👨‍🏫 老师："在你们开始之前，我提醒三个事情：

⚠️ 1. 规则最少2条，最多3条。两条就够了。

⚠️ 2. 设计图不用画得很漂亮，画出大概的样子就行。
   重要的是你的想法，不是画画的水平。

⚠️ 3. 如果不知道选什么类型——
   接东西'和'躲避'最容易完成，新手选这两个。

好，现在打开电脑，进入网站，开始你的设计！"
```

### [17-20分钟] 过渡：切换到自主实践

```
👨‍🏫 老师：（走动辅导）

重点关注：
- 前测5题是否全部完成
- 规则是否使用了「如果___，就___」句式
- 设计图是否已保存

最后3分钟提醒：
"还有3分钟，检查一下你的规则写好了吗？
规则写了的同学，检查设计图保存了吗？
都完成的同学，可以看看你的画面够不够漂亮。"
```

## 教师检查清单
- [ ] 每个学生完成前测5题
- [ ] 每个学生选择了游戏类型
- [ ] 每个学生写了≥2条规则（如果___就___）
- [ ] 每个学生保存了设计图

# 第2节：和小智老师做游戏

## 教学目标
- 学生能点击「自动生成」让AI生成游戏
- 学生能与「小智老师」至少对话 2 轮
- 学生能试玩自己的游戏并提出修改意见
- 学生能上传作品到班级

## 教学脚本（20分钟）

### [0-3分钟] 回顾：我们的设计

```
👨‍🏫 老师："上节课，你们每个人都设计了自己的游戏。
谁来展示一下你设计的是什么游戏？"

（选2个学生，让他们在投影上展示自己的设计图和规则）

👨‍🏫 老师："XX同学设计了一个接东西的游戏，规则是如果接到星星就得分。
XX同学设计了一个跑酷游戏，规则是如果撞到障碍物就扣命。

今天，我们要让AI把这些设计变成真正的游戏！"
```

### [3-8分钟] 示范：AI生成游戏

```
👨‍🏫 老师：（投影演示）
"看大屏幕。这是小智老师，一个会写代码的AI机器人。

进入「游戏设计」页面。

看到左上角的「自动生成」按钮了吗？
点一下——AI就会根据你的设计图帮你做游戏！

（点击按钮，等待10-15秒）

看！代码出来了。右边是预览区，点击「游戏」标签——
哇，游戏做好了！你可以在里面玩！

这就是AI的厉害之处：你设计，AI写代码。"
```

### [8-15分钟] 讲解：如何和AI对话修改

```
👨‍🏫 老师："游戏做出来了，但可能还不够完美。
比如我想让颜色更鲜艳，想让速度慢一点，想加个分数显示。
怎么办？——告诉小智老师就行了！

和小智老师说话就像和老师说话一样。
你们平时怎么跟老师说话，就怎么跟它说话。

举个例子：
❌ '改代码'（太模糊，AI不知道你想改什么）
✅ '颜色太暗了，改成亮一点的'（具体清楚）

❌ '不好玩'（没说哪里不好）
✅ '太简单了，敌人再快一点'（说了要改什么）

❌ 说一大堆（一次说太多AI容易搞混）
✅ 一次只说一个要改的地方

现在我来演示一下——
（现场演示：'加一个分数显示'→AI修改→预览效果）

看！每次改完都能立刻看到效果。"
```

### [15-17分钟] 理解检查：AI对话练习

```
👨‍🏫 老师："现在我们来练习一下。假设这是你刚生成的游戏，
你想让小智老师帮忙改一下，你要怎么说？

说出来——"

👦 学生："帮我加一个背景音乐"
👨‍🏫 老师："可以！很具体。还有呢？"

👧 学生："敌人移动太慢了，加速两倍"
👨‍🏫 老师："非常好！说了具体要改的东西和方向。"

👦 学生："画面太丑了，搞好一点"
👨‍🏫 老师："这个说法不够清楚哦。试试说具体要改什么？
比如：'背景颜色改成蓝色'或者'主角画大一点'"
```

### [17-20分钟] 过渡：布置任务

```
👨‍🏫 老师："好，现在自己操作。任务清单：

1️⃣ 点击「自动生成」生成你的游戏
2️⃣ 试玩一遍，找出1个想改的地方
3️⃣ 和小智老师说你要改什么（至少说2次）
4️⃣ 改好后，给游戏起个名字，点「上传」

⚠️ AI生成需要十几秒，请耐心等待。
⚠️ 如果AI回复了什么你看不懂的内容，举手问我。
⚠️ 保存时记得给游戏起名字！
"
```

## 教师检查清单
- [ ] 每个学生生成了游戏
- [ ] 每个学生至少和AI对话 2 轮
- [ ] 每个学生上传了作品（游戏名+代码）
- [ ] 游戏可以正常运行

# 第3节：游戏测评师

## 教学目标
- 学生能试玩 3 个同学的游戏
- 学生能用选项按钮给出具体评价
- 学生能查看自己收到的评价

## 教学脚本（20分钟）

### [0-3分钟] 导入：测评师的角色

```
👨‍🏫 老师："你们有没有玩过游戏之后，特别想告诉别人哪里好玩、哪里不好玩？
今天你们就是游戏测评师——就像B站上的游戏UP主一样！

你们的任务是：玩3个同学的游戏，然后告诉他们——
哪里做得好、哪里可以更好。

为什么要评价别人？
因为玩游戏的人才知道游戏好不好玩。
你给的建议，能帮助同学把游戏做得更好！
而且你也会收到3个同学给你的建议哦。"
```

### [3-8分钟] 示范：如何正确评价

```
👨‍🏫 老师：（打开一个游戏）
"评价一个游戏，先玩，再回答三个问题。

第一步：先玩！
点击「试玩游戏」，好好玩一遍。

第二步：回答三个问题。
看右边，有三个问题：
1. 哪里最好玩？（点选项）
2. 有什么建议？（点选项）
3. 发现什么问题？（点选项）

如果选项里没有你想要的——
可以在输入框里自己写。

例子：
如果我觉得这个游戏画面很好看，我就点'画面好看'
如果我觉得应该加音乐，我就点'加点音乐音效'
如果我觉得没有问题，问题那道就点'没有问题'

⚠️ 要点：
- 一定要具体，不要只说'好玩'或'不好玩'
- 要友善，你的评价同学会看到的
- 三个问题都要回答"
```

### [8-13分钟] 讲解：如何善用反馈

```
👨‍🏫 老师："评价别人很重要，看看别人怎么评价你的——同样重要。

评价完3个游戏后，点击「查看我的评价」，
你就能看到同学对你的评价了。

但是要记住：
✅ 同学说好玩的——继续保留
✅ 同学建议改的——认真考虑要不要改
✅ 同学发现问题的——优先修复

假如你收到这样的评价：
'好玩之处：画面很好看'
'建议：加个分数显示'
'问题：我按了键盘没反应'

你知道下节课应该怎么改吗？"
👦 学生："加分数！修键盘！"
👨‍🏫 老师："对了！下节课我们就根据这些建议来改。"
```

### [13-17分钟] 互动：讨论一个好评价

```
👨‍🏫 老师："我给你们看两个评价，你们觉得哪个更好？

评价A：好玩，加油。
评价B：画面很漂亮，特别是星星的设计。我建议加个声音，
      因为我接到星星的时候没有什么感觉。

哪个更好？为什么？"

👧 学生："B更好，因为说清楚了哪里好。"
👦 学生："B还说怎么改进。"

👨‍🏫 老师："对！好的评价有三个特点：
1. 具体的——说清楚了哪里好
2. 有用的——给了改进建议
3. 友好的——像在对朋友说话

你们今天的评价也要做到这三点。"
```

### [17-20分钟] 过渡：布置任务

```
👨‍🏫 老师："开始吧！任务清单：

1️⃣ 进入「同伴互评」
2️⃣ 试玩第1个游戏 → 评价 → 提交
3️⃣ 试玩第2个游戏 → 评价 → 提交
4️⃣ 试玩第3个游戏 → 评价 → 提交
5️⃣ 切换到「查看我的评价」，看看别人给你的建议

全部完成后举手，我来看一下你的评价质量。"
```

## 教师检查清单
- [ ] 每个学生完成 3 个评价
- [ ] 评价内容不是空的
- [ ] 评价内容不是乱填的（如"asdf"、"111"）
- [ ] 学生查看了自己收到的评价

# 第4节：我的游戏我做主

## 教学目标
- 学生能根据同伴评价改进游戏
- 学生能完成自我反思（Q1-Q5）
- 学生能浏览全班作品并点赞

## 教学脚本（20分钟）

### [0-3分钟] 导入：设计迭代

```
👨‍🏫 老师："还记得上节课你们收到的同学评价吗？
打开「查看我的评价」，看看同学说了什么。

今天我们要做三件事：
1. 根据同学的建议改进游戏
2. 写一个反思，回顾整个过程
3. 去看看全班同学的游戏，给你喜欢的点赞

最后，老师会选出被点赞最多的3个游戏，让大家在投影上展示！"
```

### [3-9分钟] 示范：反思填写

```
👨‍🏫 老师：（投影演示）
"进入「我的反思」，系统已经帮你填好了一些内容。

看我这里——游戏名称和规则已经自动填好了！
这是根据你第一节课的设计自动填的，你可以修改。

Q1：描述你的游戏
'我的游戏叫[太空大战]，玩法是[控制飞船消灭敌人]。'
（已自动填入，检查是否正确）

Q2：说明你的规则
'如果[子弹打中敌人]，就[得一分]。'
（已自动填入，检查是否正确）

Q3：遇到的困难——这里要你自己填。
下面有很多选项，你点一个就行。
比如：'代码太复杂写不出来'、'不知道怎么改更好'
然后写你是怎么解决的：'问AI老师帮忙'、'自己多试了几次'

Q4：同伴的反馈——这里也要你自己填。
看看上节课同学给你的评价，选一个。
比如：'同学给了我很好的建议'、'还需要继续改进'
然后写你的感受：'很开心有人喜欢'、'觉得还有进步空间'

Q5：如果重新做——你会怎么改？
比如：'修改游戏规则'、'把画面做得更好看'

每个问题点选项+补充填空就行了，很简单。
你一分钟就能搞定！"
```

### [9-14分钟] 讲解：根据反馈修改

```
👨‍🏫 老师："现在回到「游戏设计」。
看看同学给你什么建议——

如果是'加点音乐音效'→和AI说'加背景音乐'
如果是'调整速度'→和AI说'速度改慢一点'
如果是'加得分显示'→和AI说'加得分显示'

原则：一次只改一个地方！改完看看效果，再改下一个。

你现在有8分钟时间修改。
修改完后上传最新的版本。"
```

### [14-18分钟] 展示：班级作品

```
👨‍🏫 老师："大家都改好了吗？
现在进入「班级作品」——

这里能看到全班同学的游戏！
你可以：
- 浏览游戏卡片，看缩略图
- 点击你喜欢的一个进去试玩
- 觉得好的，点右下角的❤️点赞

给你4分钟，去看看同学的游戏，给你喜欢的点赞！

谁的点赞最多，我们就请谁来展示。"
```

### [18-20分钟] 成果展示

```
👨‍🏫 老师："好，时间到。

现在我们来看看——

⭐ 点赞最多的是谁？（展示排行榜前3名）

请XX同学来展示你的游戏！（投影展示）

👨‍🏫 老师："XX同学，你这个游戏最满意的地方是哪里？"
👦 学生："最满意的是画面，我觉得很好看。"
👨‍🏫 老师："如果再来一次，你会想改什么？"
👦 学生："我想加个Boss关。"

👨‍🏫 老师："非常好！4节课下来，你们每个人都做出了自己的游戏。
从设计到制作，从评价到改进——
你们已经是真正的游戏设计师了！🎉"
```

## 教师检查清单
- [ ] 每个学生根据反馈修改了游戏
- [ ] 每个学生完成了反思（Q1-Q5都有内容）
- [ ] 每个学生浏览了至少1个同学的游戏
- [ ] 班级作品有被点赞
